## 2026-03-04 - [High] End-to-End Encryption masking XSS payload evaluation
**Vulnerability:** XSS (Cross-Site Scripting) vulnerability where the client-side code used `.innerHTML` to insert decrypted messages directly into the DOM (e.g. `<b>nome</b>`).
**Learning:** End-to-End Encryption (E2EE) at the transport/server layer does not protect against malicious payloads sent by authenticated peers. Since the client directly trusted and rendered the decrypted string as HTML, any user could broadcast a malicious XSS payload which would decrypt properly and execute on other clients. Combining cryptographic security features can sometimes lead developers into a false sense of security regarding input sanitization.
**Prevention:** Never use `.innerHTML` or string interpolation with raw HTML tags to render dynamically received content. Always treat decrypted payloads as untrusted user input. Use safe DOM manipulation methods such as `document.createElement()` and `textContent`, and transmit data as structured JSON objects instead of pre-formatted HTML strings.
