## 2026-03-02 - [CRITICAL/HIGH] Fix Cross-Site Scripting (XSS) vulnerability
**Vulnerability:** The client logic concatenated HTML tags strings with unescaped user input and rendered it on the page directly utilizing `.innerHTML` after decrypting it (`document.getElementById("chatBox").appendChild(document.createElement("p")).innerHTML = messageChatBox;`). This allowed malicious users to execute arbitrary JavaScript (XSS) by including `<script>` or other event handler payloads inside their messages which would then be executed upon receiving the decrypted message on the client's side.
**Learning:** Even when the connection encrypts traffic using end-to-end encryption, you cannot trust user input. Client-side rendering of decrypted messages must still enforce HTML sanitization.
**Prevention:** Always use safe DOM APIs like `.textContent` and `.createElement` to construct elements, or rely on a standard sanitization library. Never use `.innerHTML` to insert text inputs without explicit and secure sanitization.
